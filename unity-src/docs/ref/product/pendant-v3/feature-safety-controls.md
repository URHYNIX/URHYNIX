## 정지 계열 버튼 배치

### TopStatusBar 제어 영역
```text
┌─ 제어 버튼 ────────────────────────────────┐
│                                             │
│  [서보ON]  [▶시작]  [■ 정지]  [⏸ 일시정지] │
│                     ▲ 빨간배경   ▲ 주황배경  │
│  [↻Sync]  [⟲오류초기화]                   │
│                     ▲ 빨간텍스트             │
│                                             │
│  ■ 정지: 항상 가장 눈에 띄는 위치            │
│  ■ 정지 버튼은 다른 버튼보다 크게 (56x44px) │
│                                             │
└─────────────────────────────────────────────┘
```

### BottomBar 제어 (중복 배치)
```text
┌─ BottomBar ────────────────────────────────────┐
│  [▶실행] [■정지] [▷DryRun] [Step◀] [Step▶]   │
│           ▲ 빨간배경                            │
│           ■ 정지는 TopBar + BottomBar 양쪽 배치 │
└────────────────────────────────────────────────┘
```

> **설계 근거**: 정지 버튼은 모든 실제 펜던트에서 가장 접근성 높은 위치에 있음.
> 소프트 펜던트에서도 화면 어디를 보고 있든 정지를 누를 수 있어야 함.

## 안전 상태별 UI 반응

### 1. 정상 상태
```text
┌─ TopStatusBar ─────────────────────────┐
│  [안전: ● 정상]                        │
│  배경: 기본 (--rc-backdrop)             │
└────────────────────────────────────────┘
```

### 2. 경고 상태 (작업공간 한계 접근)
```text
┌─ TopStatusBar ─────────────────────────┐
│  [안전: ⚠ 주의]                       │
│  배경: AccentWarning 5%               │
│  + 하단 경고 배너 슬라이드 다운         │
└────────────────────────────────────────┘

┌─ 경고 배너 (48px) ────────────────────┐
│  ⚠ J3 관절이 한계(225°)에 근접합니다  │
│  현재: 218.0° / 한계: 225.0°          │
│                            [무시] [정지]│
└────────────────────────────────────────┘
```

### 3. Fault 발생
```text
┌─ TopStatusBar ─────────────────────────┐
│  [안전: ⛔ FAULT]                      │
│  배경: AccentDanger 10%               │
└────────────────────────────────────────┘

┌─ Fault 배너 (전체 너비) ──────────────┐
│  ⛔ 에러 발생: 충돌 감지               │

│  모든 조작이 잠겼습니다.               │
│  [원인 보기]  [오류 초기화]            │
└────────────────────────────────────────┘
```

## 위험 방지 UX 패턴

### 이동 전 위험 감지
```text
┌─ 이동 확인 다이얼로그 ───────────────────┐
│                                          │
│  ⚠ 이동 경로에 위험 요소가 있습니다      │
│                                          │
│  ┌─ 위험 요약 ──────────────────────┐   │
│  │  • 특이점 근접 (J5 = 89.5°)      │   │
│  │  • 이동 거리가 큽니다 (342mm)     │   │
│  │  • 속도 30%에서 약 2.3초 소요     │   │
│  └──────────────────────────────────┘   │
│                                          │
│  [취소]  [DryRun으로 확인]  [이동 실행]  │
│                                          │
└──────────────────────────────────────────┘
```

### 위험 수준별 UI 반응

| 수준 | 색상 | 동작 |
|------|------|------|
| 안전 | 없음 | 바로 실행 가능 |
| 주의 | `AccentWarning` 테두리 | 경고 메시지 표시, 실행 가능 |
| 위험 | `AccentDanger` 테두리 | 확인 다이얼로그 필수 |
| 차단 | `AccentDanger` 배경 | 실행 버튼 비활성화, 사유 표시 |

### 이동 방향 색상 변화
- 안전한 방향으로 이동: 조그 버튼 기본 색상
- 한계 방향으로 이동: 조그 버튼 `AccentWarning` 테두리
- 한계 초과 방향: 조그 버튼 비활성화

## E-Stop 안내 (소프트 펜던트 한계)

> 소프트 펜던트에는 물리 E-Stop이 없다.
> 따라서 다음 규칙을 따른다:

1. **첫 실행 가이드**에서 물리 E-Stop 위치 안내
2. **TopStatusBar에 E-Stop 안내 아이콘** 상시 표시
3. **소프트 정지(■)는 E-Stop이 아님**을 명확히 표시
4. 소프트 정지 = `ImmStopJOG` + `ProgramStop` (감속 정지)
5. 실제 E-Stop은 물리 펜던트 또는 외부 스위치에서만 가능

## 팝업 트리거 규칙 (U4, U10)

| 상황 | 팝업 여부 | 팝업 내용 |
|------|----------|----------|
| MoveJ/MoveL 실행 (Live) | **필수** | "실제 로봇이 움직입니다" |
| 서보 ON | **필수** | "서보를 활성화합니다" |
| 에러 초기화 | **필수** | "에러를 초기화합니다" |
| 포인트 삭제 | **필수** | "정말 삭제하시겠습니까?" |
| 시퀀스 삭제 | **필수** | "이 시퀀스를 삭제하시겠습니까?" |
| 수정 중 탭 전환 | **필수** | "저장하지 않은 변경이 있습니다" |
| 수정 중 창 닫기 | **필수** | "저장하지 않은 변경이 있습니다" |
| 일반 탭 전환 | 없음 | - |
| 조그 모드 전환 | 없음 | - |
| 좌표계 전환 | 없음 | - |
| DryRun 이동 | 없음 | - |

### 팝업 UI 규칙
- 동시 최대 1개 (이전 닫힌 후 새 팝업)
- 버튼 순서: 왼쪽=취소(`MutedText`), 오른쪽=확인(`Accent` 또는 `Danger`)
- 외부 클릭: 경고/확인=무시, 도움말=닫기

### 위험 버튼 설명 규칙 (사용자 피드백 반영)

위험하거나 결과가 큰 버튼은 색상만으로 의미를 전달하지 않는다.
각 버튼은 실행 전 아래 3가지를 보여줘야 한다.

1. 이 버튼이 **무엇을 하는지**
2. 실제 로봇이 움직이는지, 상태만 바뀌는지
3. 취소 가능 여부

| 버튼 | 필수 사전 설명 | 취소 가능 여부 |
|------|------|------|
| `MoveJ / MoveL 실행` | "실제 로봇이 움직입니다." | 팝업에서 취소 가능 |
| `서보ON` | "모터를 활성화해 이동 가능 상태로 바꿉니다." | 팝업에서 취소 가능 |
| `정지` | "현재 동작을 감속 정지합니다." | 누른 뒤 되돌리기 불가, 이후 재시작 필요 |
| `오류초기화` | "Fault를 해제하고 상태를 다시 읽습니다." | 팝업에서 취소 가능 |
| `포인트 삭제` | "저장된 포인트를 목록에서 제거합니다." | 팝업에서 취소 가능 |

- 초보자 모드에서는 버튼 라벨 아래 또는 우측 보조 문구를 기본 노출한다.
- 전문가 모드에서는 hover/focus/long-press 설명으로 축약할 수 있다.
- `정지` 버튼만은 항상 "E-Stop 아님" 문구를 접근 가능한 위치에 같이 둔다.
