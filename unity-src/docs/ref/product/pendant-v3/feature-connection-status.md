### 연결 상태 인디케이터
```text
┌───────────────────────────────────────────────────┐
│ 미연결 상태:                                       │
│   ○ 미연결   [모드:--]  [속도:--]  [안전:--]      │
│                                                    │
│ 연결됨/서보OFF:                                    │
│   ● 연결됨   [모드:수동] [속도:30%] [안전:정상]   │
│   ⚠ 서보 OFF                                     │
│                                                    │
│ 정상 운전:                                         │
│   ● 연결됨   [모드:수동] [속도:30%] [안전:정상]   │
│   ✓ 서보 ON                                      │
│                                                    │
│ 에러 상태:                                         │
│   ● 연결됨   [모드:--]   [속도:--]  [⛔ FAULT]    │
│   전체 배경: AccentDanger 10%                     │
└───────────────────────────────────────────────────┘
```

### 제어 버튼 배치
```text
┌────────────────────────────────────────────────┐
│ [서보ON]  [▶시작]  [■정지]  [⏸일시정지]       │
│ [↻동기화] [⟲오류초기화]                       │
│                                                │
│ 버튼 상태 규칙:                                 │
│ - 미연결: 모든 버튼 비활성                      │
│ - 서보OFF: 서보ON만 활성                        │
│ - 서보ON: 시작/동기화 활성                      │
│ - 실행중: 정지/일시정지 활성                    │
│ - Fault: 오류초기화만 활성                      │
└────────────────────────────────────────────────┘
```

### 카드 내 상태별 색상

| 필드 | 정상 | 경고 | 위험 |
|------|------|------|------|
| 연결 | `AccentSuccess` 녹색 점 | - | `AccentDanger` 빨간 점 |
| 모드 | `TitleText` 흰색 | `AccentWarning` (Drag) | - |
| Fault | `MutedText` "없음" | - | `AccentDanger` 코드 표시 |
| Safety | `AccentSuccess` "정상" | `AccentWarning` | `AccentDanger` "정지" |

### 다음 행동 카드 규칙 (사용자 피드백 반영)

연결 홈의 `다음 행동 추천`은 단순 도움말이 아니라, 현재 상태에서 **사용자가 가장 먼저 해야 할 1개 행동**만 강조한다.

| 현재 상태 | Primary Next Action | 보조 설명 |
|------|------|------|
| 미연결 | `연결` | "먼저 로봇과 연결해야 현재 상태를 읽을 수 있습니다." |
| 연결됨 / 서보 OFF | `서보 켜기` | "서보가 켜져야 로봇을 움직일 수 있습니다." |
| 서보 ON / 미동기화 | `동기화` | "현재 실제 로봇 상태를 먼저 가져오세요." |
| 동기화 완료 / 첫 진입 | `쉬운 조작 시작` | "Home 또는 Ready로 먼저 작은 동작을 확인하세요." |
| Fault | `원인 보기` | "원인을 먼저 확인한 뒤 초기화를 진행하세요." |
| 자동 재연결 중 | `기다리기` | "지금은 재연결을 시도 중입니다. 수동 연결은 잠시 뒤 가능합니다." |

- 카드에는 항상 `지금 상태`, `다음 행동`, `왜 이걸 먼저 해야 하는지` 3가지를 같이 표시한다.
- 같은 단계에서 선택지가 여러 개 있어도 primary CTA는 1개만 강조한다.
- 사용자가 이미 필요한 조건을 만족했으면 다음 단계 CTA로 즉시 교체한다.

### 버튼 결과 설명 규칙

TopStatusBar와 연결 홈의 위험도 있는 버튼은 hover, focus, long-press 또는 보조 라벨을 통해 **실행 결과**를 먼저 설명해야 한다.

| 버튼 | 필수 설명 문구 |
|------|------|
| `서보ON` | "로봇 모터를 활성화합니다. 이후 실제 이동이 가능해집니다." |
| `시작` | "현재 선택한 동작 또는 시퀀스를 실행합니다." |
| `정지` | "현재 동작을 감속 정지합니다. 물리 E-Stop은 아닙니다." |
| `일시정지` | "현재 실행을 멈추고, 이후 재개할 수 있습니다." |
| `동기화` | "실제 로봇의 현재 자세와 상태를 다시 읽습니다." |
| `오류초기화` | "Fault를 해제하고 다시 조작 가능 상태로 되돌립니다." |

## 자동 재연결 (A2)

```text
연결 끊김 감지
  → 상태 바: "연결 끊김 — 재연결 시도 중 (3초)"
  → 3초 간격 자동 재시도
  → 카운트다운 표시: "3... 2... 1... 재시도"
  → 최대 10회 시도 후 "자동 재연결 실패. [수동 연결]"
  → 재연결 성공 시 토스트: "연결이 복원되었습니다"
```

## 초보자 친화 요소

1. **상태 전이 시 토스트 알림**: "서보가 켜졌습니다", "동기화 완료"
2. **Fault 발생 시 풀스크린 배너**: 빨간 배경, 큰 글씨, 해결 순서 표시
3. **미연결 시 조작 패널 마스킹**: 반투명 오버레이 + "먼저 연결하세요" 메시지
4. **DryRun 기본 ON**: Live 모드 첫 연결 시 DryRun 자동 활성화
5. **현재 단계 라벨**: `연결`, `준비`, `동기화`, `조작 가능`, `오류 처리 중` 중 하나를 항상 표시
