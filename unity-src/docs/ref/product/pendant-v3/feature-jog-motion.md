## 조작 보조패널 내부 탭 구조

```text
좌측 NavRail [조작] 클릭
└─ ViewportHost 보조패널
   ┌──────────────────────────────┐
   │ [기본] [관절] [TCP] [좌표]   │
   │   ▲                          │
   │ 기본 선택                    │
   └──────────────────────────────┘
```

- `WorkTabBar`는 더 이상 메인 상단의 독립 탭처럼 보이지 않는다.
- `조작` 좌측 탭이 active일 때만 `ControlDockHost` 첫 줄에 내부 조작 모드로 표시한다.
- `포인트`, `I/O`, `상태`, `도움` active 상태에서는 이 내부 조작 탭을 숨긴다.
- 라벨은 operator 부담을 줄이기 위해 `기본 / 관절 / TCP / 좌표`로 잠근다.
- `좌표`는 기존 `포인트 이동` 기능이며, 좌측 `포인트` 탭의 티칭 포인트 저장/시퀀스/함수와 분리한다.

## 1. 기본 탭

### 용도
- 초보자가 처음 접할 때 큰 버튼으로 안전한 프리셋 포즈로 이동
- 숙련자도 자주 쓰는 포즈를 빠르게 호출
- **그리퍼 열기/닫기** 버튼 포함 (U1)

### 와이어프레임
```text
┌─ 기본 ──────────────────────────────────────┐
│                                               │
│  프리셋 포즈                                   │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐      │
│  │   🏠    │  │   ✋    │  │   📦    │      │
│  │  Home   │  │  Ready  │  │  Folded │      │
│  │ (원점)  │  │ (대기)  │  │ (접힘)  │      │
│  └─────────┘  └─────────┘  └─────────┘      │
│  ┌─────────┐                                  │
│  │   ⭕    │                                  │
│  │  Zero   │                                  │
│  │ (영점)  │                                  │
│  └─────────┘                                  │
│                                               │
│  ───────────────────────────────────────────  │
│                                               │
│  실행 옵션                                     │
│  ☑ 미리보기 먼저 (DryRun)                     │
│  속도: [━━━●━━━━━] 30%                        │
│                                               │
│  [미리보기]          [실제 이동]              │
│  (3D에서 확인)       (로봇 구동)              │
│                                               │
└───────────────────────────────────────────────┘
```

### V3 compact 보조패널 규칙
- Desktop V3에서는 관절 조그가 `ViewportHost` 보조패널 안에서 보인다.
- 가로 스크롤은 금지한다.
- 각 관절 행은 좁은 폭 기준으로 3단으로 쌓는다.
  - `J축 + 입력 + 현재값`
  - `슬라이더 + 관절 한계`
  - `J- / J+`
- 값/입력/버튼이 오른쪽에서 잘리면 패널을 키우지 말고 행 내부 폭을 줄인다.

### 버튼 사이즈
- 프리셋 버튼: **최소 88x88px** (태블릿 터치 최적)
- 실행 버튼: **최소 120x44px**

## 2. 관절 조그 탭

### 용도
- 개별 관절을 독립적으로 조작
- 다축 연계 슬라이더 (FAIRINO 스타일)
- 단일축 long-press 조그 (FANUC 스타일)

### V3 compact 보조패널 규칙
- Desktop V3에서는 TCP 조그와 3D 방향 조작이 `ViewportHost` 보조패널 공용 스크롤 안에서 보인다.
- 가로 스크롤은 금지한다.
- `X/Y/Z/RX/RY/RZ` 행은 한 줄 고정이 아니라 2줄 구조다.
  - 상단: `축 + 값 + 단위`
  - 하단: `- / +`
- `Z / RX / RY / RZ` 조작 UI는 메인 `RobotStage` 위에 올라오지 않는다.
- 보조 툴바 텍스트는 `Base / Tool / Path / Ghost / Bound / Coll / Cam`처럼 짧게 유지한다.

### 인풋 UX 규칙 (U5)
- TextField `FocusInEvent` → `SelectAll()` — 0을 지우지 않아도 바로 입력
- 입력 즉시 미리보기 반영 (`RegisterValueChangedCallback`, Enter 불필요)
- NaN/Infinity 입력 시 즉시 거부 + 이전값 복원 (`SetValueWithoutNotify`)

### 슬라이더 인터랙션
- **드래그**: 목표값 변경 (로봇 안 움직임, 미리보기만)
- **[▶]/[◀] 버튼**: 증분만큼 ±
- **[입력]**: 숫자 직접 입력 (NaN/Infinity 검증)
- **[복원]**: 현재 실제 로봇 값으로 슬라이더 리셋
- **[미리보기]**: 3D에서 고스트로 목표 자세 표시
- **[적용]**: 실제 MoveJ 실행 (확인 다이얼로그 → DryRun 또는 Live)

### 관절 한계 표시
- 슬라이더 양 끝에 한계값 표시
- 한계 근접 시(90% 이상) 슬라이더 색상 `AccentWarning`으로 전환
- 한계 초과 시 입력 거부 + 토스트 "관절 한계를 초과했습니다"

## 3. TCP 조그 탭

### 용도
- TCP(Tool Center Point)를 좌표계 기준으로 직선/회전 이동
- Base/Tool/User 좌표계 전환

### 와이어프레임
```text
┌─ TCP 조그 ───────────────────────────────────┐
│                                               │
│  좌표계: [●Base]  [ Tool]  [ User]            │
│                                               │
│  ┌─ 직선 이동 (mm) ──────────────────────┐   │
│  │                                        │   │
│  │     [━━ X- ━━]  -497.0  [━━ X+ ━━]   │   │
│  │     [━━ Y- ━━]  -130.0  [━━ Y+ ━━]   │   │
│  │     [━━ Z- ━━]   477.0  [━━ Z+ ━━]   │   │
│  │                                        │   │
│  └────────────────────────────────────────┘   │
│                                               │
│  ┌─ 회전 이동 (°) ───────────────────────┐   │
│  │                                        │   │
│  │     [━ RX- ━]   180.0°  [━ RX+ ━]    │   │
│  │     [━ RY- ━]     0.0°  [━ RY+ ━]    │   │
│  │     [━ RZ- ━]    90.0°  [━ RZ+ ━]    │   │
│  │                                        │   │
│  └────────────────────────────────────────┘   │
│                                               │
│  증분: [0.1] [1] [●5] [10] [50] mm/°         │
│  속도: [━━━●━━━━━] 30%                        │
│                                               │
│  ※ 길게 누르면 연속 이동                      │
│  ※ 손 떼면 즉시 정지                          │
│                                               │
└───────────────────────────────────────────────┘
```

### 좌표계 전환 동작
- **Base**: 로봇 베이스 기준 XYZ 이동 — 초보자 기본
- **Tool**: 현재 공구 방향 기준 이동 — 공구 끝 정밀 작업
- **User**: 사용자 정의 좌표계 — 작업대 기준 이동
- 전환 시 3D 뷰포트에 해당 좌표축 하이라이트 (RGB: X=Red, Y=Green, Z=Blue)

### +/- 버튼 인터랙션 (FAIRINO 원형버튼 + Doosan 방향버튼 참고)
- **탭**: 증분만큼 1회 이동
- **길게 누르기 (300ms 이상)**: 연속 이동 시작
- **손 떼기**: 즉시 정지 (`ImmStopJOG` 호출)
- **버튼 사이즈**: 최소 56x44px (터치 영역)

### 데카르트 3D 화살표 방향 조작 (U12)
- 3D 뷰포트에 XYZ 화살표 오버레이 표시
- 화살표 클릭/드래그 → 해당 방향으로 TCP 이동
- 화살표 색상: X=Red, Y=Green, Z=Blue (좌표축과 일치)
- 회전 링: RX/RY/RZ 도넛 형태 오버레이

### 3D 뷰포트 연동
- TCP 조그 탭 활성화 시 3D에서 좌표축 + 화살표 표시 자동 ON
- 이동 방향에 해당하는 축 강조 (나머지 축 반투명)

## 4. 좌표 탭

### 용도
- 목표 좌표를 직접 입력하고 IK 계산 후 이동
- 저장된 포인트 불러와서 이동

### 와이어프레임
```text
┌─ 좌표 ──────────────────────────────────────┐
│                                               │
│  목표 입력 방식: (●) 좌표 입력  ( ) 포인트    │
│                                               │
│  ┌─ 좌표 입력 ───────────────────────────┐   │
│  │  X: [________] mm                      │   │
│  │  Y: [________] mm                      │   │
│  │  Z: [________] mm                      │   │
│  │  RX: [________] °                      │   │
│  │  RY: [________] °                      │   │
│  │  RZ: [________] °                      │   │
│  │                                        │   │
│  │  [현재값 채우기]  [포인트에서 불러오기] │   │
│  └────────────────────────────────────────┘   │
│                                               │
│  ┌─ IK 결과 ─────────────────────────────┐   │
│  │  도달 가능: ✓ 예                       │   │
│  │  예상 관절값:                           │   │
│  │  J1: 12.3°  J2: -45.0°  J3: 67.8°    │   │
│  │  J4: 0.0°   J5: 90.0°   J6: -12.5°   │   │
│  │  ⚠ 특이점 근접: J5 = 90°             │   │
│  └────────────────────────────────────────┘   │
│                                               │
│  이동 방식: (●) MoveJ   ( ) MoveL             │
│  속도: [━━━●━━━━━] 30%                        │
│                                               │
│  [미리보기]          [이동 실행]              │
│                                               │
└───────────────────────────────────────────────┘
```

### IK 사전 검증
- 좌표 입력 즉시 IK 계산 → 도달 가능성 표시
- 도달 불가: `AccentDanger` + "이 위치에 도달할 수 없습니다"
- 특이점 근접: `AccentWarning` + 어떤 관절이 위험한지 표시
- 정상: `AccentSuccess` + 예상 관절값 표시

## 공통 안전 규칙

1. **모든 이동 명령은 미리보기 → 확인 → 실행** 흐름
2. **Live 모드**: 확인 다이얼로그 필수 ("실제 로봇이 움직입니다. 진행하시겠습니까?")
3. **DryRun 모드**: 확인 없이 3D에서만 동작
4. **Mock 모드**: 확인 없이 3D에서만 동작
5. **Fault 상태**: 모든 이동 버튼 비활성화
6. **미연결 상태**: 모든 탭 마스킹
