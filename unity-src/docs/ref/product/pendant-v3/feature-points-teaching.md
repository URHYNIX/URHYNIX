# 포인트 저장/호출 + 티칭 시퀀스

## Purpose
- 웨이포인트 저장, 이름 관리, 목록, 시퀀스 편집/재생의 UI를 정의한다.
- 현장 작업의 핵심 흐름: "점 찍고 → 다시 가고 → 반복하기"

## SSOT 상태
- **SSOT 확정**: V3 PointMove v1에서 save/recall/list/delete/rename/export/cleanup이 wired 상태다.
- 저장소는 기존 `WaypointStore`를 재사용하며 V3 sequence 이름은 `PendantV3Points`다.
- saved `MoveJ` single-point live apply와 `PendantV3Points` two-point live run-once는 current branch에서 green이다.
- 현재 production gap은 `broad named sequence live`와 `live loop`를 여는 것이지, `PendantV3Points` one-shot 자체가 아니다.
- 전체 실행 계획은 [teaching-sequence-execution-plan.md](./teaching-sequence-execution-plan.md)를 SSOT로 본다.
- 현재 실기 성공패턴은 [fr5-teaching-point-live-success-pattern.md](../roadmap/fr5-teaching-point-live-success-pattern.md)를 SSOT로 본다.
- 확장 방향은 새 왼쪽 `Program` 탭이 아니라 기존 `NavPoints` 내부의 `포인트 / 시퀀스 / 함수` subview다.

## Navigation Expansion Policy

- 왼쪽 메인 Nav는 현재 `NavPoints`를 유지한다.
- `NavPoints`의 의미는 "포인트 저장"에서 "티칭 포인트 + 실행 시퀀스"로 확장한다.
- 제조사 Lua/program 기능과 혼동되는 별도 `Program` left-nav는 만들지 않는다.
- 상용 티칭펜던트식 확장은 `NavPoints` 내부 subview로 나눈다.
  - `포인트`: 저장, 호출, 조회, 수정, 삭제, 묶음 후보 선택/라이브러리 저장.
  - `묶음`: 저장된 함수 라이브러리 조회/이름 변경/복사/삭제/일괄 관리.
  - `시퀀스`: 순서 조정, Step, Run, Stop, Loop, 묶음 호출.

## Ease-Of-Use Policy

- 기존 상용 티칭펜던트보다 쉬운 흐름을 목표로 한다.
- 사용자는 position register, program node, script 용어를 몰라도 포인트를 저장하고 실행할 수 있어야 한다.
- 포인트 상세는 반드시 사람이 읽기 쉬워야 한다.
  - 이름
  - MoveJ/MoveL
  - Joint
  - TCP
  - speed
  - dwell
- 상용 pendant의 touch-up은 `현재 위치로 덮어쓰기`로 제공한다.
- `Run From Selected`는 `선택 지점부터 실행`으로 표현한다.
- 함수화는 `포인트` subview에서 선택 포인트를 묶어 `함수 라이브러리`로 저장하는 흐름을 우선한다.

## Locked V1 Behavior

- 포인트 이름은 사용자 입력이다.
- 빈 이름은 저장하지 않고 명확한 안내를 보여준다.
- 중복 이름은 확인 후 기존 포인트를 덮어쓴다.
- 덮어쓰기는 기존 순서를 유지한다.
- `현재 위치로 덮어쓰기`는 `jointsDeg`와 `tcpMm`만 바꾸고 `moveType`, `speedPreset`, `dwellSec`는 유지한다.
- 기본 저장 moveType은 `MoveJ`다.
- v1 저장 기준 좌표계는 Base다.
- 실행 중에는 삭제, 순서 변경, 이름 변경, 덮어쓰기, 복사를 잠근다.
- 시퀀스 실행 중 실패하면 즉시 정지하고 실패한 포인트를 선택 상태로 남긴다.
- 포인트 목록은 사용자 순서를 유지하며 이름순 자동 정렬을 하지 않는다.

## Point / Bundle / Sequence Linkage Policy

- `포인트`, `묶음`, `시퀀스`는 서로 독립 화면이 아니라 같은 티칭 데이터의 세 관점이다.
- `포인트`에서 저장한 `PendantV3Points`는 `시퀀스` 탭에서 항상 선택/실행 가능한 기본 시퀀스로 보여야 한다.
- `기록 중지`로 생성되는 `PendantV3RecordedPath`도 시퀀스 라이브러리에 보여야 하며, 사용자는 이를 재생/루프/삭제할 수 있어야 한다.
- `시퀀스` 탭은 최소한 다음 시퀀스를 구분해 보여준다.
  - `PendantV3Points`: 사용자가 저장한 티칭 포인트 순서
  - `PendantV3RecordedPath`: 연속 경로 기록 결과
  - 기타 `WaypointStore`에 저장된 named sequence
- `묶음` 탭은 새 묶음을 만드는 곳이 아니라 저장된 함수 라이브러리를 관리하는 곳이다.
- 묶음 생성은 `포인트` 탭에서 선택 포인트를 묶어 `함수 라이브러리 저장`으로 끝낸다.
- `묶음` 탭은 선택한 포인트 참조를 묶는 기능이 아니라, 이미 저장된 묶음의 참조/누락 상태와 관리 액션을 보여준다.
- 탭 순서는 `포인트 / 묶음 / 시퀀스`다.
- `시퀀스` 탭의 `작업 시퀀스`는 `PointRef`와 `BundleRef` 블록을 한 줄짜리 실행 블록으로 섞어 저장한다.
- 작업 시퀀스 저장소 이름은 `PendantV3Blocks`다.
- 삭제 정책:
  - `PendantV3Points` 삭제는 포인트 전체 삭제와 같은 위험 작업으로 보고 기존 cleanup/confirmation 흐름을 유지한다.
  - `PendantV3RecordedPath`와 기타 recorded/named sequence는 시퀀스 탭에서 삭제 가능해야 한다.
  - 삭제 후 현재 선택된 시퀀스가 사라졌다면 `PendantV3Points`를 기본 선택으로 복귀한다.
- 구현 기준:
  - `NavPoints > 시퀀스`는 `저장한 포인트 순서`, `기록한 경로`, `실행 목록`을 같은 화면에 보여준다.
  - `NavPoints > 시퀀스`는 별도 `함수 라이브러리` 목록도 같이 보여주고, 묶음을 바로 `작업 시퀀스`에 추가할 수 있어야 한다.
  - `작업 시퀀스`는 포인트 블록과 묶음 블록을 추가/삭제/위아래 이동/미리보기/실행할 수 있다.
  - 실행 목록 row는 `선택 / 재생 / 루프 / 삭제`를 제공한다.
  - 실행 목록 row의 `선택` 버튼으로 여러 시퀀스를 선택하고, 삭제 가능한 항목만 일괄 삭제할 수 있다.
  - `버튼 접기`는 실행 목록 row를 `선택 / 열기` 중심으로 줄인다.
  - `PendantV3Points` row의 삭제는 비활성화하고, 포인트 탭의 개별 삭제/정리 confirmation으로만 처리한다.
  - `기록 삭제` 버튼은 `PendantV3RecordedPath` 삭제 confirmation으로 연결된다.
  - v1 row `1회 재생`은 current branch에서 live gate + popup confirm 경로로 허용된다.
  - v1 row `루프`는 아직 Unity/Mock 검증 범위에 가깝고 live 실기 실행은 계속 잠근다.
- 묶음 row UX 기준:
  - 묶음 탭 상단 요약은 함수 라이브러리 개수, 선택 개수, 선택 묶음의 참조/누락 정보를 보여준다.
  - 묶음 row의 `선택` 버튼으로 여러 묶음을 선택할 수 있다.
  - 일괄 작업은 `선택 해제`, `선택 복사`, `선택 삭제`, `전체 삭제`를 제공한다.
  - `전체 삭제`는 confirmation 2회 클릭으로만 실행한다.
  - 묶음 탭 자체는 생성/실행 탭이 아니라 라이브러리 관리 탭으로 본다.
  - 선택 묶음의 이름 변경, 복사, 삭제는 row/하단 액션으로 처리한다.
  - 묶음 저장소는 `Application.persistentDataPath/teaching-functions`이며, 전체 삭제는 이 저장소의 모든 묶음 JSON을 제거한다.
  - `버튼 접기`는 묶음 row를 `선택 / 열기` 중심으로 줄인다.
- 포인트 row UX 기준:
  - row 전체 클릭은 선택/상세 확인만 한다.
  - row의 `선택` 버튼으로 여러 포인트를 선택할 수 있다.
  - row 버튼은 `실행`, `미리보기`, `편집`, `묶음 추가`로 표시한다.
  - row 버튼은 즉시 스크롤을 요구하지 않고, 먼저 포인트 작업 모달을 연다.
  - `편집` 모달은 이름, 속도, dwell, 현재 위치 덮어쓰기, 복사, 삭제를 한 화면에서 처리한다.
  - `현재 위치 저장`은 `포인트` 탭에서만 보여야 한다.
  - 상단 요약은 포인트 개수, 함수 개수, 속도별 개수, 선택 개수를 보여준다.
  - 일괄 작업은 `선택 해제`, `선택 묶음 후보`, `선택 삭제`를 제공한다.
  - 포인트 목록 아래에는 `함수 라이브러리로 저장` 카드가 있어야 하며, 후보 요약/이름 입력/후보 초기화/저장을 한 자리에서 처리한다.
  - 함수 등록 소스 우선순위는 `묶음 후보 -> 현재 보고 있는 포인트 -> 선택 포인트 -> 전체 저장 포인트` 순서로 단순하게 이해돼야 한다.
  - 묶음 후보가 비어 있으면 `현재 보고 있는 포인트 1개`를 바로 함수 등록하는 fallback이 있어야 한다.
  - 사용자는 별도 `묶음 탭`으로 넘어가기 전에 `포인트` 탭에서 바로 함수 등록을 끝낼 수 있어야 한다.
  - `버튼 접기`는 row를 `선택` 중심으로 줄여 긴 목록 스캔을 쉽게 만든다.
  - 기본 안내/feedback 문구는 줄이고 confirmation, 저장, 삭제, 실패, bulk 결과만 짧게 보여준다.

- 포인트 속도/대기 UX 기준:
  - 기존 detail 카드의 속도/대기 직접 조절 UI는 제거한다.
  - 속도/대기 수정은 `편집` modal에서 포인트별로만 다룬다.
  - 사용자는 row `편집`을 눌렀을 때만 속도/대기 값을 건드릴 수 있어야 한다.

- 시퀀스 묶음 추가 UX 기준:
  - `시퀀스 > 작업 시퀀스 > 묶음 추가`는 즉시 추가가 아니라 `묶음 라이브러리 modal`을 연다.
  - modal에서 사용자가 원하는 함수 1개를 선택하고 `시퀀스 추가`로 확정한다.
  - actual click 기준으로 `묶음 추가 -> modal open -> 시퀀스 추가 -> BundleRef block 생성`이 검증되어야 한다.

### 스텝 컨텍스트 메뉴 [⋮]
- 위로 이동 / 아래로 이동
- 복제
- 수정
- 삭제

### 시뮬레이션 vs 실행
- **시뮬레이션**: 3D에서만 시퀀스 재생 (실제 로봇 안 움직임)
- **실행**: Live 모드에서 실제 로봇 구동 (확인 다이얼로그)

## 블록 에디터 (U2, U11, U13)

### 블록 종류 + 아이콘 매핑 (U3)

**이동 명령**
| 블록 | 아이콘 | 설명 |
|------|--------|------|
| MoveJ | 곡선 화살표 | 관절 기준 이동 |
| MoveL | 직선 화살표 | 직선 이동 |
| MoveC | 원호 화살표 | 원호 보간 이동 (U13) |

**IO 명령**
| 블록 | 아이콘 | 설명 |
|------|--------|------|
| 그리퍼 열기 | 열린 손 | Open |
| 그리퍼 닫기 | 닫힌 손 | Close |
| DO ON/OFF | 번개 | 디지털 출력 |

**논리 명령 (U11)**
| 블록 | 아이콘 | 설명 |
|------|--------|------|
| Wait | 시계 | N초 대기 |
| Loop | 순환 | N회 반복 (시작~끝 범위 지정) |
| IF | 분기 | 조건 분기 (DI 상태 기준) |
| Call | 호출 | 다른 시퀀스 호출 |
