# Project Plan

## Sprint 1: 기획 / 기본 화면

- MVP 범위와 팀 역할 확정
- Unity 경기장 blockout
- 관제 UI 기본 틀 제작
- 트랙/장애물 소품 준비

## Sprint 2: ROS / 카메라 입력

- TurtleBot3 SLAM/Nav2 기본 주행 확인
- Unity-ROS pose 연동
- 카메라 설치와 스트림 확인
- 카메라 프레임과 로봇 pose 동기화

## Sprint 3: 장애물 / Vision / Unity 표시

- LiDAR 장애물 감지와 회피/정지 기준 구현
- 카메라 기반 실시간 장애물 인식
- 장애물 클래스와 촬영 데이터셋 구축
- Unity에 카메라 화면과 인식 결과 표시
- LiDAR 장애물 테스트 환경 세팅

## Sprint 4: DB / QA / Demo

- 주행 데이터와 인식 결과 저장
- 카메라 이미지와 인식 로그 저장 구조 확장
- LiDAR only vs Camera + Vision 비교 데모
- 카메라 인식 주행 시나리오 검증
- 발표용 화면/영상 준비

## 추천 진행 순서

1. 기획 확정
2. 경기장/트랙 고정
3. TurtleBot3 SLAM/Nav2 확인
4. Unity pose 표시
5. 카메라 스트림 확인
6. 카메라 pose 동기화
7. Vision 인식
8. Unity 표시
9. DB 저장
10. 발표용 검증
