# PRD

## 한 줄 소개

Unity Digital Twin 관제 화면에서 TurtleBot3의 자율주행, 장애물 인식, 카메라 인식 결과를 확인하는 프로젝트입니다.

## 왜 만들까?

움직이는 로봇이 주변 장애물을 어떻게 인식하고 대응하는지 일반인도 화면으로 이해할 수 있게 만들기 위해서입니다.

## 누가 쓸까?

- 로봇·자율주행·이미지 인식·디지털트윈을 배우는 학생과 개발자
- 재난 대응 로봇을 연구하는 사람 (소방, 채굴장, 미탐사지역)
- 자율주행/로봇 데이터를 분석하는 사람
- 로봇 관제 시스템 개발자
- 카메라 인식과 LiDAR 기반 장애물 대응을 비교하고 싶은 사람
- 로봇 시연을 보는 일반 관람자

## 이번에 만들 것

- TurtleBot3 SLAM/Nav2 기반 주행
- LiDAR 기반 장애물 감지와 정지/회피
- 카메라 기반 장애물 인식
- Unity Digital Twin 관제 UI
- 주행 결과와 인식 결과 DB 저장
- 발표용 비교 데모: LiDAR only vs Camera + Vision

## 이번에는 안 만들 것

- FR5 로봇팔 제어
- 픽앤플레이스
- 장애물 물리 제거
- 완전 자동 재난 구조 시나리오

## 성공 기준

- v1: TurtleBot3가 SLAM/Nav2로 장애물을 피해 목적지까지 이동한다.
- v2: Unity 관제 화면에서 로봇 위치·경로·장애물 상태가 실시간으로 보인다.
- v3: 카메라(또는 샘플 이미지)로 장애물 종류를 인식하고 Unity 표시 + DB 저장된다.
- v4: 두 가지 주행 조건(LiDAR only vs LiDAR + Vision)을 비교해 도착 시간·정지 횟수·인식 정확도를 표로 보여준다.
- 발표 기준: 일반인이 "로봇이 무엇을 보고 어떻게 반응했는지" 화면만 보고 이해할 수 있다.

## 데이터 수집 전략

단순 시연이 아니라 "AI 학습용·분석용 데이터셋"을 함께 수집한다. 발표·취업 포트폴리오 핵심 무기.

| 필드 | 설명 |
|---|---|
| `Trial_ID` | 주행 회차 |
| `Mode` | `lidar_only` 또는 `lidar_vision` |
| `Obstacle_Layout` | 그 회차 트랙 장애물 배치 |
| `Total_Time` | 출발 → 목적지 시간(초) |
| `Stop_Count` | 정지 횟수 |
| `Avg_Speed` | 평균 속도 |
| `Detection_Accuracy` | 실제 장애물 vs 인식 결과 일치율 |

시연 한 번이 아니라 누적된 비교 데이터가 발표 핵심. 가능하면 무한 루프 시뮬레이션으로 데이터 양 확보.

## 발표 포인트

1. **"단순 자율주행이 아닌, 판단 과정 시각화"** — Unity 디지털트윈으로 로봇이 무엇을 보고 어떻게 판단했는지 실시간 표시.
2. **"두 조건 비교 데이터셋"** — LiDAR only vs LiDAR+Vision 누적 비교로 정량적 결과 제시.
3. **"안전 우선 설계"** — Vision은 보조, LiDAR가 최종 안전 판단 (현실 적용 가능성 강조).
4. **"확장 가능한 구조"** — 재난·물류·실내 안내 등 확장 시나리오.

## 알려진 리스크

| 리스크 | 대응 |
|---|---|
| 카메라 인식 정확도가 낮음 | S3에서 데이터셋 보강, 실패 시 샘플 이미지 fallback |
| ROS-Unity 통신 지연 | S2 초반에 bridge 부담 측정, 필요시 토픽 다운샘플링 |
| TurtleBot3 하드웨어 문제 | 박태진이 S1~S2에 사전 확인, 백업 부품 확보 |
| 시연 환경 광량 변화 | S4 직전 시연 장소에서 카메라 사전 캘리브레이션 |
| 일정 지연 | S3 종료 시점에 v4(비교 데모) 단순화 가능 여부 검토 |
