# Demo Validation Checklist

## Demo Scenarios

- [ ] 빈 경로에서 목적지까지 이동한다.
- [ ] 정면 장애물을 감지하고 정지 또는 회피한다.
- [ ] 측면 장애물을 카메라가 인식한다.
- [ ] LiDAR only 조건과 Camera + Vision 조건을 비교한다.

## Unity Screen

- [ ] 로봇 현재 위치가 보인다.
- [ ] 이동 경로가 보인다.
- [ ] 장애물 상태가 보인다.
- [ ] 카메라 이미지 또는 캡처가 보인다.
- [ ] 인식 class/confidence가 보인다.

## DB / Evidence

- [ ] 주행 시간 기록이 남는다.
- [ ] 정지 횟수 기록이 남는다.
- [ ] 인식 결과 기록이 남는다.
- [ ] 발표용 캡처 또는 영상이 있다.
- [ ] 실패했을 때 보여줄 replay/영상이 있다.

## Presentation

- [ ] 한 줄 소개가 준비되어 있다.
- [ ] 왜 만들었는지 설명할 수 있다.
- [ ] 각 팀원 역할을 설명할 수 있다.
- [ ] 성공 기준을 화면/데이터로 보여줄 수 있다.
